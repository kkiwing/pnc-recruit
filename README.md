# P&C 채용관리 (P&C Recruitment Manager)

## 1. 프로젝트 개요

P&C(인사)팀의 채용 프로세스를 관리하기 위한 웹 애플리케이션 **프로토타입**입니다. 채용 공고 등록부터 지원자 접수, 전형 단계 진행 상황, 면접 일정까지 한 화면에서 관리하는 흐름을 다룹니다.

현재 버전은 실제 백엔드 없이 동작하는 **순수 프론트엔드 인메모리 프로토타입**입니다. 앱을 새로고침하면 모든 데이터가 `src/data/` 아래의 더미 데이터로 초기화됩니다.

## 2. 목적

이 저장소는 프로덕션에 배포할 서비스가 아니라, **테크팀이 실제 개발에 착수할 수 있도록 화면 흐름과 데이터 구조를 미리 검증해보는 기획용 프로토타입**입니다. P&C팀이 원하는 채용 관리 화면/기능/데이터 모델을 먼저 프론트엔드로 구현해보고, 이를 바탕으로 테크팀과 요구사항을 정렬한 뒤 실제 백엔드가 있는 서비스로 재구축하는 것을 목표로 합니다.

## 4. 주요 기능

- **공고 관리**: 채용 공고 등록/수정/삭제, 진행중/종료 상태 자동 판정(마감일 기준), 공고별 지원 현황 요약
- **프로세스 관리**: 공고마다 전형 단계를 자유롭게 구성(추가/삭제/순서 변경, 상태값 커스터마이징). 새 공고는 관리 가능한 "기본 프로세스 프리셋"을 복사해서 시작하며, 이후 프리셋을 바꿔도 이미 만들어진 공고에는 소급 적용되지 않습니다.
- **지원자 관리**: 지원자 등록/수정/삭제, 목록/파이프라인(칸반) 두 형태로 조회, 단계별 상태를 인라인으로 변경
- **면접 일정**: 캘린더·타임테이블, 예정/지난(결과 미입력)/완료 3분류
- **별도 관리**: 미응시·중도포기 등의 지원자를 별도로 모아 관리하고 필요 시 복귀
- **지원자 메모/이력 관리**: 지원자별 특이사항 메모, 첨부파일 업로드(현재는 브라우저 세션 내 임시 저장)

각 기능의 정확한 동작 조건과 예외 케이스는 [functional-spec.md](docs/functional-spec.md)를 참고하세요.

## 5. 기술 스택

- **프레임워크**: React 18 + Vite 5 + TypeScript
- **UI**: shadcn/ui (Radix 기반), Tailwind CSS, lucide-react
- **라우팅**: react-router-dom
- **상태 관리**: React Context + `useState` (인메모리, 백엔드 연동 없음)
- **테스트**: Vitest + Testing Library (단위), Playwright(e2e 설정은 있으나 현재 미사용)
- **패키지 매니저**: npm

> `@tanstack/react-query`, `react-hook-form` + `zod`, `recharts`는 설치되어 있지만 현재 화면에서는 사용하지 않습니다. 실제 백엔드 연동 시 데이터 페칭/폼 검증/차트 용도로 활용을 고려해볼 수 있습니다.

## 6. 로컬 실행 방법

```bash
npm install
npm run dev
```

기본적으로 `http://localhost:8080`에서 실행됩니다 (포트가 사용 중이면 다른 포트로 자동 전환).

기타 스크립트:

```bash
npm run build     # 프로덕션 빌드
npm run lint       # ESLint 검사
npm run test       # Vitest 단위 테스트 실행
```

## 7. 환경변수 설정 위치

**현재는 별도의 환경변수가 필요하지 않습니다.** 이전에는 Supabase 연동을 위한 `.env`가 있었으나, 순수 인메모리 프로토타입으로 전환하면서 제거했습니다.

향후 실제 백엔드를 연동하게 되면, 프로젝트 루트에 `.env` 파일을 생성해 `VITE_` 접두사가 붙은 값들을 정의하면 Vite가 자동으로 읽어들입니다. `.env`는 `.gitignore`에 포함되어 있으므로 커밋되지 않습니다 — API 키나 시크릿을 코드에 직접 넣지 마세요.

## 8. 프로토타입의 한계와 실제 구축 시 참고할 점

데이터 비영속, 인증/권한 없음, 파일 첨부 비영속, 면접 평가표·오퍼/입사 관리·알림 발송 미구현, 변경 이력 없음 등 이 프로토타입의 한계와, 서버 스키마 설계·CRUD 로직 재구현 시 눈여겨봐야 할 지점은 **[functional-spec.md 4장](docs/functional-spec.md#4-실제-구축-시-권장사항)**에 정리되어 있습니다.
