# 프로젝트 개요

## 1. 프로젝트 개요

**P&C 채용관리 시스템**은 P&C(인사)팀이 사내 채용 프로세스를 관리하기 위한 내부 도구입니다. 채용 공고 등록, 지원자 접수, 전형 단계별 진행 상황 관리, 별도관리(미응시·중도포기 등) 처리, 면접 일정 관리를 하나의 화면 흐름 안에서 처리하는 것을 목표로 합니다.

현재 버전은 **기획 단계 프로토타입**입니다. 실제 백엔드/DB 없이 브라우저 메모리에서만 동작하는 순수 프론트엔드로 구현되어 있으며, 새로고침하면 모든 데이터가 초기 시드 데이터로 리셋됩니다.

## 2. 목적

이 저장소는 프로덕션에 배포할 서비스가 아니라, **테크팀이 실제 개발에 착수하기 전에 화면 흐름과 데이터 구조를 검증해보는 기획용 프로토타입**입니다. P&C팀이 원하는 채용 관리 화면·기능·데이터 모델을 먼저 프론트엔드로 구현해보고, 이를 바탕으로 테크팀과 요구사항을 정렬한 뒤 실제 서비스로 재구축하는 것을 목표로 합니다.

작업 맥락(의사결정 배경, 남은 작업, 인수인계 정보)이 대화 기록에만 남지 않도록 `docs/` 아래 문서로 관리합니다.

- [decision-log.md](decision-log.md) — 날짜별 주요 의사결정과 배경
- [todo.md](todo.md) — 진행 상황과 다음 작업 우선순위
- [handover.md](handover.md) — 신규 참여자를 위한 인수인계 문서
- [DESIGN_SYSTEM.md](DESIGN_SYSTEM.md) — UI 디자인 기준

## 3. 현재 범위

| 화면 | 설명 |
|---|---|
| 대시보드 | 공고 현황(진행중/전체/합격 등) 요약, 공고별 현황 테이블 |
| 공고 관리 | 채용 공고 목록(검색/필터/정렬), 등록/수정/삭제, 공개 여부 토글 |
| 공고 상세 | 공고 기본 정보 + 본문 + 자기소개서 문항 + 전형 프로세스 요약을 보여주는 허브 화면. 지원자 목록/별도 관리/프로세스 관리로 이동하는 바로가기 제공 |
| 프로세스 관리 | 공고별 전형 단계를 자유롭게 구성(추가/삭제/순서 변경), 단계 유형(일반/합불 판정) 지정, 완료 처리 방식(입력폼) 설정, 단계별 상태값(이름/색상/의미) 커스터마이징, 자동 발송 문구 설정(발송 자체는 미구현) |
| 지원자 목록 | 목록형 테이블과 파이프라인(칸반) 뷰를 토글하여 조회, 검색/필터/정렬, 전형 단계·상태 인라인 변경 |
| 지원자 상세 | 기본 인적사항, 학력/자격증/경력/활동/통계 패키지/논문, 자기소개서, 메모, 첨부파일(임시 저장) |
| 별도 관리 | 미응시·중도포기 등 사유로 별도관리 처리된 지원자 목록, 사유 필터, 지원자 목록으로 복귀 |
| 면접 일정 | 면접 캘린더 + 날짜별 타임테이블, 예정/완료 면접 목록 |

화면 간 이동은 공고를 기준으로 연결됩니다. 지원자 목록·별도 관리·프로세스 관리는 URL 쿼리(`?posting={공고ID}`)로 특정 공고가 선택된 상태로 진입할 수 있고, 그 반대로 각 화면에서 공고 상세로 돌아가는 링크도 제공합니다.

## 4. 주요 기능

- **공고 CRUD**: 등록/수정/삭제, 검색·필터(게시 상태/고용형태/팀)·정렬
- **전형 단계 커스터마이징**: 공고마다 다른 전형 단계 구성 가능. 단계 유형(일반/합불 판정)과 상태값 의미(완료/합격/불합격)를 명시적으로 지정해 합불 판정·완료 여부를 순서나 이름 문자열이 아닌 명시적 플래그로 판별
- **지원자 관리**: 등록/수정/삭제, 이름·이메일·연락처·메모 검색, 단계·상태·팀 필터
- **두 가지 조회 방식**: 지원자를 테이블(목록형) 또는 칸반(파이프라인)으로 전환하여 확인
- **별도 관리**: 미응시/중도포기 등 사유를 기록하고 별도로 관리, 필요 시 목록으로 복귀
- **면접 일정 관리**: 완료 처리 시 입력한 일정 정보를 캘린더·타임테이블에 자동 집계
- **디자인 시스템 적용**: 라이트 모노크롬 톤의 통일된 UI, 색상·그림자·라운딩 토큰 기반 (`DESIGN_SYSTEM.md` 참고)

## 5. 기술 스택

- **프레임워크**: React 18 + Vite 5 + TypeScript
- **UI**: shadcn/ui(Radix 기반) + Tailwind CSS + lucide-react
- **라우팅**: react-router-dom (쿼리 파라미터 기반 화면 간 연결 포함)
- **상태 관리**: React Context + `useState` — 순수 인메모리, 백엔드 연동 없음
- **테스트**: Vitest + Testing Library (현재 예시 테스트 수준, 커버리지 낮음)
- **패키지 매니저**: npm

`@tanstack/react-query`, `react-hook-form` + `zod`, `recharts`는 설치는 되어 있으나 현재 화면에서는 사용하지 않습니다. 실제 백엔드 연동 시 데이터 페칭/폼 검증/차트 용도로 활용을 고려할 수 있습니다.

## 6. 1차 MVP에 포함할 기능

실제 서비스 구축 시 1차로 포함해야 할 핵심 기능입니다.

- 채용 공고 등록/수정/삭제 및 조회
- 지원자 등록/조회/검색 및 상세 정보 관리
- 공고별 커스터마이징 가능한 전형 단계 관리와 지원자별 단계·상태 진행
- 별도 관리(미응시/중도포기 등) 처리 및 목록 복귀
- 면접 일정 캘린더/타임테이블 집계
- 공고를 중심으로 한 화면 간 이동(지원자 목록/별도 관리/프로세스 관리 연결)
- 채용 담당자 인증/권한 (현재 프로토타입에는 없으나 실 서비스 1차 범위에는 필수)
- 데이터 영속 저장(실제 DB 연동)

## 7. 1차 MVP 제외 범위

다음 항목은 1차 구축 범위에서 제외하고, 향후 별도 논의 후 범위를 정합니다.

- 외부 공개 채용 사이트 생성 기능
- 지원자 직접 지원/인증 기능
- 브랜드 사이트 디자인 기능
- 실제 알림/메일·문자 발송 연동 (현재는 발송 문구를 설정하는 화면만 존재)
- 오퍼/입사 온보딩 프로세스(처우 협의, 입사일 확정 등)
- 전형 단계 변경 이력(감사 로그) 추적
- 면접 평가표(스코어카드) 등 정량 평가 도구
